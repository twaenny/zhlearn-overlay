## Talking Points

- „**5-State-Pill-Modell** matched die Spec exakt (Bid §4.3): Angemeldet → Warteliste → Abgemeldet → Teilgenommen → No-Show. Status-Wechsel sind admin-correctable, nicht nur user-driven."
- „**Add / Move / Remove + Status-Change** sind Row-Actions — Silvia kann z.B. einen Teilnehmer aus Warteliste auf Angemeldet ziehen, ohne das ganze System zu täuschen."
- „**Export `.xlsx` + `.csv`**: TS04 verlangt das für jede List-View. Format-Auswahl explizit, nicht Default-Download."
- „**Anwesenheits-Spalte** ist bewusst platzhaltend — wir sagen ehrlich, dass GoShow-Capture in der Tablet-App passiert (Bid §4.3 'keine TS')."
- „**Bulk-Mail-Aktion** (Composer-Modal) ist Teil des nächsten Sub-Points — dieselbe Page, aber als 3.2 separat fokussiert."
