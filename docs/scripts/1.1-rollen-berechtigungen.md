## Talking Points

- „Das **Berechtigungsmodell ist systemseitig fix** — Admins können keine neuen Permissions erfinden, das wäre Governance-Risiko. Was sie tun: bestehende Rollen Nutzenden zuweisen, mit einem **Scope** (Direktion, Amt, OE)."
- „9 kanonische Rollen, im Bid §2 spezifiziert — Admin, Superuser, ZHlearn-Manager, Führungsperson, Externe Kursleitende, Interne User, Externe User, Analytics, Service Desk."
- „Der **Scope** ist 1st-class — eine Rolle kann gleichzeitig auf 3 Direktionen wirken, oder Amt-genau eingeschränkt sein. Smart-Chip-Display („3 Direktionen") hält die Liste lesbar."
- „Jede Zuweisung erzeugt einen **Audit-Trail**. Wer hat wann was zugewiesen, wann ist die nächste Review fällig — das ist der Standard fürs Datenschutz-Reporting nach IDG ZH."
- Übergang: „Schauen wir uns an, was passiert, wenn jemand zwei Rollen gleichzeitig hat — das ist KTZH-spezifisch und Sie werden's so nirgends anders sehen."
