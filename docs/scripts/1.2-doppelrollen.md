## Talking Points

- **(Climax-Satz, einmal langsam:)** „Das ist der Punkt, an dem ein Standard-LMS scheitert. Eine Person kann gleichzeitig Mitarbeiterin in zwei Direktionen sein — z.B. Frau Winzeler arbeitet 60% im Personalamt der Finanzdirektion und 40% als Analytics-Verantwortliche der Volkswirtschaftsdirektion. Beide brauchen unterschiedliche Berechtigungen, unterschiedliche Pflichtangebote, unterschiedliche Vorgesetzte. Wir modellieren das **getrennt**, statt sie in eine künstliche Hauptrolle zu zwingen."
- „**Mehrfachanstellung** = 1 Person, mehrere ANSTELLUNGNR. **Doppelrolle** = 1 Anstellung, mehrere Rollen. Beides kommt vor."
- „Visuelle Konventionen: **2×-Pille** in der Liste, **Context-Card pro Anstellung** im Detail, **„Doppelrolle erkannt" Banner** wenn kritische Kombo (z.B. Admin + Analytics → braucht Datenschutz-Review)."
- „Rollenwechsel: Admin kann **als User auftreten** (Bid §2.4) — wichtig fürs Debugging + Rollen-spezifisches Testen."

## Bid-Compliance-Evidence

- **§3 Bid:** *„Anzeige von Nutzenden mit Doppelrollen und/oder Mehrfachanstellung"* — direkt erfüllt durch r3 + r4.
